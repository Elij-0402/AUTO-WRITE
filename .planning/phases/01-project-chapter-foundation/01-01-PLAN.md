## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → IndexedDB | Client-side data storage, no network transport. Data stays local. |
| User Input → UI | Project titles, synopses, and genre selections are user-provided text rendered in the UI. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | S | ProjectCard title display | accept | No authentication in Phase 1 — single user, local device. Spoofing not applicable until Phase 8. |
| T-01-02 | T | IndexedDB data | accept | Client-side only, no transport. Tampering requires local device access. |
| T-01-03 | I | Project synopsis display | mitigate | Sanitize user input before rendering (React's JSX auto-escapes by default, but verify no dangerouslySetInnerHTML usage). |
| T-01-04 | D | IndexedDB availability | mitigate | Implement graceful error handling when IndexedDB is unavailable (private browsing, storage full). Show user-friendly Chinese error message. |
| T-01-05 | E | Soft delete elevation | accept | No privilege levels in Phase 1. Deletion is user-initiated with confirmation. |
</threat_model>
