# CEO Plan: InkForge v0.3 — Sharpen the Spine

Generated by /plan-ceo-review on 2026-04-20
Branch: main | Mode: SELECTIVE EXPANSION
Repo: Elij-0402/AUTO-WRITE (InkForge)

## Context

v0.2 已经落地一个功能丰富的 AI 写作工作台（多 provider + RAG + citations + A/B + revisions + sync + analysis + export）。问题不是功能少，是每个功能是否还在为一句可测量的产品承诺服务。

本次审视选 SELECTIVE EXPANSION：基线承认 v0.2，逐条 cherry-pick 能尖锐化差异的扩张。

## Vision

**一句产品承诺（可测量版本）**：
> "深夜写 1 万字时，人物不跑偏、情节不撞车。"

所有工程决定回答：这如何让作者更快发现或避免一致性问题？

### Sharpen-the-Spine Vision

把 InkForge 从 "AI 写作工具 with 世界观" 转成 "世界观驱动的写作伙伴"。差异化 = citations 可交互 + contradiction 有历史 + 氛围 *感觉* 不同。

## Step 0E / Section 决策预决（不可逆但已拍板）

- **T1 reason = enum**（conflict / style / plot / other）+ 可选 free-form 备注
- **T3 contradictions = 新表**（schema v12）
- **T3 contradictions 同步云端** — 顺序：Supabase migration → Dexie v12 → 部署
- **T4 迁移策略**：老 `analyses.style` 只读不反向填；WorldEntry.inferredVoice 用 `{ aiDraft, userEdit? }` 双栏
- **T2 citation** 用 popover（Radix）；drawer 预留给"5+ citations"未来扩展
- **T6 深夜模式 = 扩展现有 focusMode**（不引入第二个 mode）；per-session localStorage
- **T3 入口**：编辑器顶栏按钮（主） + 分析页 tab + sidebar 角标（三处并存）
- **T5 不接 CI**（本地 `test:e2e`）；6 scenarios（5 core + 1 stress）
- **T8 快捷键 Ctrl+Alt+S**（S=Stats，避 Chrome 冲突）
- **use-ai-chat.ts 三拆**：`use-chat-stream` / `use-chat-telemetry` / `use-chat-agent`（单测必带）
- **Dexie quota sweep 并入 T1**：`safeTableWrite` 封装 + 90 天 aiUsage 清理 + UI 横幅
- **citation 点击埋点并入 T2**：`aiUsage.kind = 'citation_click'`
- **DESIGN.md 加 audio 区块**：tick 默认关 + -30dB + 仅"章节完成"1 次

## Accepted Scope (本次 v0.3 周期)

- **[T1] Draft Adoption Telemetry**
  - `aiUsage` 表加字段：`draftOffered: boolean`、`draftAccepted: boolean`、`draftEditedPct: number | null`、`draftRejectedReason: string | null`
  - 聊天消息有 `hasDraft` 时 UI 暴露"插入到编辑器"+"不采纳"两个按钮
  - 不采纳时弹出三选一 radio（"与世界观有冲突" / "文风不对" / "情节不对"）
  - 采纳后 30 分钟内，通过 `revisions` diff 推算 editedPct

- **[T2] Interactive Citation Chip**
  - citation chip 点击弹出小卡片展示该 WorldEntry 名称 + 首 2 行描述
  - 卡片里有"跳到条目"按钮 → 切 left-sidebar 到 world tab + 选中该 entry
  - 符合 DESIGN.md：朱砂边、6px 圆角、fade-in 150ms、无 shadow、无 pulse

- **[T3] Contradiction Dashboard**
  - 新视图：放在 analysis 页内加一个 tab，或编辑器顶栏加 drawer（二选一见 UI 设计时再定）
  - 每条历史矛盾：时间 + 涉及 WorldEntry + AI 描述 + 豁免状态 + 点击可跳转章节
  - 聚合：按 WorldEntry 统计矛盾次数，高频的在世界观侧栏条目后加角标

- **[T4] Style Fingerprint Repositioning**
  - 从分析页的"作者文风摘要"下架
  - 新位置：WorldEntry 编辑表单底部加"AI 推断的口吻/风格特征"section（仅 character/location 类型）
  - 把 `analyses` 表的 style 数据 migrate 到 WorldEntry 的新字段（或查询时 join）
  - 未来可以作为一致性检查的维度（AI 生成时对比 WorldEntry 的风格特征）

- **[T5] Playwright Core Path E2E**
  - 场景 1：创建项目 + 创建章节 + 编辑器保存（Dexie 持久化）
  - 场景 2：创建 3 个 WorldEntry + 刷新 + 确认仍在
  - 场景 3：打开 AI 聊天 + 发送（mock Anthropic stream） + 收到 citation chip + 点击 chip 跳转
  - 场景 4：contradiction 卡片出现 + 豁免 + 刷新 + 确认仍豁免
  - 场景 5（可选）：导出 EPUB + 验证内容完整

- **[T6] 三更书房氛围层**
  - 10 分钟无输入 → topbar 渐暗至 60% 不透明度；任何 keydown/mousemove 恢复
  - 顶栏加《深夜模式》toggle（月亮图标）→ 隐藏右 AI 面板 + 隐藏左 sidebar，只留编辑器
  - 章节状态从 `draft` 切到 `completed` 时触发一次低频 tick（60ms sine burst，-30dB，可全局关）

- **[T7] RAG 增量索引**
  - `embeddings` 表查 `lastIndexedAt` → 仅对 `worldEntries.updatedAt > lastIndexedAt` 重新嵌入
  - 首次 index 仍全量（无向量）
  - 在 `src/lib/rag/indexer.ts` 的 `indexWorldEntries` 里加条件分支
  - 单测：empty index、全更新、无更新、部分更新 4 case

- **[T8] dev-stats 隐藏面板**
  - `Ctrl+Alt+S` 触发右侧抽屉（类似 AI panel 但更窄 240px）
  - 展示：`aiUsage` 30 天聚合（total draft offered / accepted / avg editedPct / avg citations per msg / citation click rate）、`contradictions` 按 WE 聚合 top 10、indexer 最近 20 次 latency
  - BYOK 模式下这是你自己做产品判断的唯一窗口

- **[T9] 移除 GenerationDrawer（SCOPE REDUCTION 在 v0.3 内）**
  - 删除文件：`generation-drawer.tsx` / `generation-button.tsx`（components）、`use-chapter-generation.ts`（hook）、可能 `content-validator.ts`（仅该 hook 使用时）
  - 清理引用：`projects/[id]/page.tsx` 的 `GenerationDrawer` import 与 state、`ui-flags.ts` 的 `showGenerationPipeline`
  - 保留 AI 聊天中的"草稿插入"路径（T1 服务该路径）——即：AI 生成依然存在，但从"抽屉生成一整章"变成"AI 聊天建议 + 点插入"
  - 单测：相关测试文件一并删（`generation-drawer.test.tsx` 等）

- **[T10] `extendedCacheTtl` 默认开**
  - `src/lib/ai/experiment-flags.ts` 的 `DEFAULT_EXPERIMENT_FLAGS.extendedCacheTtl = true`
  - UI 保留 toggle（用户可关，理由：隐私偏好/费用担忧）
  - README 加一行说明"默认启用 1 小时 prompt cache TTL"

- **[T11] 首次引导重写**
  - 保留 onboarding AIConfigDialog 作第一步（填 API key）
  - 第二步：引导用户创建 3 个 character 类 WorldEntry（提供示例："主角" / "反派" / "师父"模板可跳过填写）
  - 第三步：引导发送第一条消息"介绍一下这 3 个角色"
  - 第四步：高亮 citation chip —— "点击这里看 AI 怎么理解你的世界观"
  - 完成后：跳转到编辑器，第一章 placeholder 已预建
  - 不强制完成（任意步骤可 skip），但路径清晰
