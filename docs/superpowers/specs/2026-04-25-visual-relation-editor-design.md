# InkForge 可视化关系编辑器 - 设计规格书

Generated by brainstorming on 2026-04-25

## 1. 概述

可视化关系编辑器是 InkForge 关系图谱的 Phase 1 功能，核心目标是让用户通过点击节点触发 AI 分析，发现潜在关系并快速建立连接。设计遵循"小白友好"原则：用户只需点击一下，AI 帮忙发现应该建立什么关系。

## 2. 核心交互流程

### 2.1 主流程

```
用户点击节点 → 节点高亮 → 右侧浮现 AI 分析面板 → AI 分析中...
    ↓
[成功] 显示推荐列表（按关系强度排序）
    ↓
用户点击推荐 → 表单预填关系类型/描述 → 用户确认 → 关系建立 → 面板关闭
    ↓
[推荐含未创建节点] 显示为虚线 → 用户点击 → 确认创建 → 自动建关系

[失败] 显示 Toast 错误 + 重试按钮 → 用户可重试或关闭面板
```

### 2.2 双击空白画布流程

```
用户双击空白区域 → 弹出快速创建浮层 → 用户填写信息 → 创建完成
    ↓
自动切换到新创建节点 → 触发 AI 分析 → 推荐面板浮现
```

## 3. AI 分析面板

### 3.1 位置与样式

- **位置**：节点右侧浮动面板，距离节点 12px
- **宽度**：280px
- **最大高度**：400px（超出滚动）
- **背景**：`surface-2`，1px `line-strong` 边，8px 圆角
- **出现动效**：`fade-up`，200ms，`cubic-bezier(0.2, 0, 0, 1)`

### 3.2 面板内容结构

```
┌─────────────────────────────┐
│ [节点图标] 节点名称           │  ← 标题栏
│ 类型：角色                   │
├─────────────────────────────┤
│ AI 分析中...                 │  ← 加载状态
│ [旋转加载动画]               │
├─────────────────────────────┤
│ 为您推荐 5 个潜在关系         │  ← 成功状态
│                             │
│ ┌─────────────────────────┐ │
│ │ 赵云     [战友]         │ │  ← 推荐项
│ │ 同为蜀国五虎将           │ │
│ └─────────────────────────┘ │
│ ┌─────────────────────────┐ │
│ │ 诸葛亮   [君臣]         │ │
│ │ 多次协同作战             │ │
│ └─────────────────────────┘ │
│ ┌─────────────────────────┐ │
│ │ ✦ 张郃   [对手]         │ │  ← 未创建节点（虚线）
│ │ 建议创建此角色           │ │
│ └─────────────────────────┘ │
├─────────────────────────────┤
│ [关闭]                      │  ← 底部操作
└─────────────────────────────┘
```

### 3.3 推荐项结构

- **已存在节点**：实线描边圆 + 节点名称 + 关系类型标签 + 简要理由
- **未创建节点**：虚线描边圆（`stroke-dasharray: 4,3`）+ "建议创建" 提示

## 4. AI 推荐逻辑

### 4.1 触发机制

- **触发时机**：用户点击节点后立即触发
- **触发前提**：无（不需要额外按钮或修饰键）
- **防抖处理**：200ms 内不重复触发（用户快速连续点击时）

### 4.2 AI 输入

```typescript
interface AIRecommendationInput {
  sourceNode: {
    id: string
    name: string
    type: WorldEntryType  // 角色/地点/规则/时间线
    description: string
    createdRelations: Array<{
      targetId: string
      targetName: string
      category: RelationCategory
    }>
  }
  allEntries: WorldEntry[]  // 全图上下文
}
```

### 4.3 AI 输出

```typescript
interface AIRecommendationOutput {
  recommendations: Array<{
    targetNode: {
      id?: string        // 已存在节点有 id
      name: string
      type: WorldEntryType
      isNew: boolean    // 是否为建议创建的新节点
    }
    suggestedRelation: {
      category: RelationCategory
      description: string
      confidence: number  // 0-1 关系强度
    }
  }>
}
```

### 4.4 排序规则

- 按 `confidence` 降序排列
- 已存在节点优先于未创建节点

## 5. 关系建立流程

### 5.1 点击已存在推荐

1. 用户点击推荐项
2. 弹出关系确认表单（预填 AI 推荐的关系类型和描述）
3. 用户可修改关系类型和描述
4. 用户点击"确认" → 关系建立 → 面板关闭 → 图谱更新

### 5.2 点击未创建推荐

1. 用户点击虚线推荐项
2. 弹出确认提示："张郃 不存在，是否创建？"
3. 用户点击"创建" → 弹出快速创建浮层（预填节点名称和类型）
4. 用户补充描述 → 创建完成 → 自动建立关系 → 面板关闭

### 5.3 表单预填内容

| 字段 | 来源 |
|------|------|
| 关系类型 | AI 推荐的 `category` |
| 关系描述 | AI 推荐的 `description` |
| 源节点 | 自动填充（不可修改） |
| 目标节点 | 自动填充（不可修改） |

## 6. 错误处理

| 场景 | 处理方式 |
|------|----------|
| AI 分析失败 | Toast 显示错误信息 + 面板内显示"重试"按钮 |
| AI 超时（> 10s） | Toast 显示"分析超时" + 重试按钮 |
| 推荐项已被删除 | 从列表中移除，不显示 |
| 关系建立失败 | 表单内显示错误，保留用户输入 |

## 9. MVP 优先级

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 节点拖拽 + 位置持久化 | LayoutSnapshot 表 |
| P0 | 双击节点 → 打开编辑浮层 | 现有 world entry 编辑 |
| P0 | 点击节点 → 触发 AI 分析 → 显示推荐 | 核心差异化 |
| P0 | 点击推荐 → 建立关系（预填表单 + 用户确认） | 闭环 |
| P1 | 双击空白 → 快速创建条目 → 自动进入 AI 分析 | 体验完善 |
| P1 | AI 分析失败 → Toast + 重试 | 错误处理 |

## 10. 非 Phase 1 范围

- AI 主动关系建议（虚线 → 用户确认）
- 引力吸附（snap to neighbor）
- 多布局保存/切换
- 移动端支持
- WebWorker 力仿真

## 11. 数据库变更

### v16 Migration: LayoutSnapshot 表

```typescript
this.version(16).stores({
  layoutSnapshots: 'id, projectId, [projectId+layoutId], nodeId',
})
```

### LayoutSnapshot Schema

```typescript
interface LayoutSnapshot {
  id: string           // nanoid
  projectId: string
  layoutId: string     // Phase 1 = 'default'
  nodeId: string       // WorldEntry.id
  x: number
  y: number
  isDefault: boolean
  updatedAt: number
}
```

### 级联删除

WorldEntry 删除时，通过 liveQuery 监听并主动清理对应的 LayoutSnapshot 记录。

## 12. 降级策略

| 场景 | 降级方案 |
|------|----------|
| force-layout 崩溃 | 环形布局 fallback |
| LayoutSnapshot 读取失败 | 使用 force-layout 默认位置 |
| saveLayoutSnapshot 失败 | retry once → console.warn |
| AI 分析失败 | Toast + 重试按钮 |
